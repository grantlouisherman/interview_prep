# Algorithms & Strategies

Algorithm design patterns and problem-solving strategies. Each page explains when to use the approach, how to recognize problems that fit, and walks through classic implementations.

| Strategy | When to Use | Typical Complexity |
|----------|------------|-------------------|
| [Greedy](greedy.md) | Locally optimal choices lead to global optimum | O(n log n) |
| [Dynamic Programming](dynamic-programming.md) | Overlapping subproblems + optimal substructure | O(n^2) or O(n * m) |
| [Divide and Conquer](divide-and-conquer.md) | Problem splits into independent subproblems | O(n log n) |
| [Backtracking](backtracking.md) | Exhaustive search with pruning | Exponential (with pruning) |
| [Sorting & Searching](sorting-and-searching.md) | Ordering data, finding elements efficiently | O(n log n) / O(log n) |
